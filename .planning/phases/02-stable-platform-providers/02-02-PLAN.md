Create `packages/streamwall/src/main/discovery/providers/twitch.ts`:

- `TwitchProvider` extends `BaseProvider`
- `platform = 'twitch'`
- `capabilities = { requiresCredentials: false, supportsLanguageFilter: false, isExperimental: false }`
- `rateLimit = { maxRequests: 10, windowMs: 60_000 }` (moderate for unofficial API)

Constants at module level:
```typescript
const TWITCH_GQL_URL = 'https://gql.twitch.tv/gql'
const TWITCH_CLIENT_ID = 'kimne78kx3ncx6brgo4mv6wki5h1ko'
```

GraphQL query for searching live channels (from research):
- Use the `SearchResultsPage_SearchResults` query with `targets: [{ index: CHANNEL }]`
- Request fields: `id, login, displayName, broadcastSettings.title, stream.id, stream.viewersCount, stream.game.name, stream.previewImageURL(width:440,height:248), stream.createdAt, stream.freeformTags.name, stream.broadcastSettings.language`
- Pagination via `$after: String` cursor, `$first: Int` (use 30 per page)

`onInit()`: no-op (no initialization needed for raw fetch)

`onSearch(query)`:
- Handle comma-separated keywords: split, trim, search each keyword separately
- For each keyword, POST to `TWITCH_GQL_URL` with headers `{ 'Client-ID': TWITCH_CLIENT_ID, 'Content-Type': 'application/json' }`
- Body: `{ query: SEARCH_QUERY, variables: { query: keyword, first: 30, after: cursor } }`
- Filter results: only include channels where `node.stream !== null` (live channels only)
- Map to `DiscoveredStream`:
  - `platform: 'twitch'`
  - `title: node.broadcastSettings.title`
  - `channelName: node.displayName`
  - `url: 'https://twitch.tv/${node.login}'`
  - `thumbnailUrl: node.stream.previewImageURL`
  - `viewerCount: node.stream.viewersCount`
  - `language: node.stream.broadcastSettings?.language ?? ''`
  - `tags: node.stream.freeformTags?.map(t => t.name) ?? []`
  - `startedAt: node.stream.createdAt`
- Paginate via `cursor` from response while `pageInfo.hasNextPage` and under 500 cap
- Deduplicate by URL using `Set<string>`
- Sort by viewerCount descending
- Console log: `[Twitch] GQL search for "${keyword}": ${count} results`
- On error, return `ProviderResult` with error. Specifically detect Client-ID rejection (401/403): `[Twitch] GQL Client-ID rejected`

`onDestroy()`: no-op

Use `node-fetch` (already in project dependencies) for HTTP requests. If `node-fetch` is not available at runtime in utility process, use global `fetch` (available in Electron's Node.js environment).

Create `packages/streamwall/src/main/discovery/providers/kick.ts`:

- `KickProvider` extends `BaseProvider`
- `platform = 'kick'`
- `capabilities = { requiresCredentials: false, supportsLanguageFilter: true, isExperimental: false }`
- `rateLimit = { maxRequests: 10, windowMs: 60_000 }` (moderate for unofficial endpoint)

Constants:
```typescript
const KICK_SEARCH_URL = 'https://kick.com/api/search'
const KICK_BROWSE_URL = 'https://api.kick.com/public/v1/livestreams'
```

`onInit()`: no-op

`onSearch(query)`:
- Handle comma-separated keywords: split, trim, search each keyword separately
- For each keyword, try unofficial search first:
  - `GET https://kick.com/api/search?searched_word=${encodeURIComponent(keyword)}`
  - Filter results to channels where `is_live === true` or equivalent field
  - Map to `DiscoveredStream`:
    - `platform: 'kick'`
    - `title`: stream title from response
    - `channelName`: channel display name or username
    - `url: 'https://kick.com/${slug}'`
    - `thumbnailUrl`: thumbnail from response
    - `viewerCount`: viewer count from response
    - `language: ''` (unofficial endpoint may not provide this)
    - `tags: []` (unofficial endpoint may not provide tags)
    - `startedAt`: stream start time if available, else `''`
- If unofficial search fails (Cloudflare block, 403, etc.), fall back to official browse API:
  - `GET https://api.kick.com/public/v1/livestreams?sort=viewer_count&limit=100`
  - This does NOT support keyword search, so results will be the top livestreams (not filtered by keyword)
  - Log: `[Kick] Search endpoint unavailable, falling back to browse API (no keyword filtering)`
- Deduplicate by URL using `Set<string>`
- Sort by viewerCount descending
- Cap at 500 total
- Console log: `[Kick] Search for "${keyword}": ${count} results`

Per CONTEXT.md: "Best effort: implement with whatever endpoints work; if they break, return empty results with error status."

The Kick API response format may differ from what the research documented. During implementation, inspect actual response shapes and adapt mapping accordingly. If both endpoints fail, return `{ streams: [], error: { type: 'unavailable', message: '[Kick] All endpoints failed' } }`.

`onDestroy()`: no-op
