  <action>
1. Install youtubei.js:
   ```bash
   cd packages/streamwall && npm install youtubei.js
   ```

2. Create `packages/streamwall/src/main/discovery/providers/youtube.ts`:
   - Single `YouTubeProvider` class extending `BaseProvider`
   - `platform = 'youtube'`
   - `capabilities = { requiresCredentials: false, supportsLanguageFilter: true, isExperimental: false }`
   - `rateLimit = { maxRequests: 5, windowMs: 60_000 }` (conservative for Innertube per research)
   - `onInit()`: call `Innertube.create()` and store instance. Handle failure gracefully (log error, set innertube to null)
   - `onSearch(query)`:
     - Check `getApiKey('youtube')` -- if present, delegate to `searchViaDataApi(query, apiKey)`, otherwise `searchViaInnertube(query)`
     - Handle comma-separated keywords: split by comma, trim, run separate searches per keyword
     - Deduplicate results by URL using a `Set<string>`
     - Sort by viewerCount descending
     - Cap at 500 total results (stop paginating once reached)
     - Return `ProviderResult` with streams array
   - `searchViaInnertube(query)`:
     - `this.innertube!.search(query, { features: ['live'] })`
     - Map results to `DiscoveredStream[]` with `platform: 'youtube'`, `url: 'https://youtube.com/watch?v=${video.id}'`
     - Paginate via `search.getContinuation()` while `has_continuation` and under 500 cap
     - If `viewerCount` not available from search results, set to 0
     - Console log: `[YouTube] Innertube search for "${query}": ${count} results`
   - `searchViaDataApi(query, apiKey)`:
     - `GET https://www.googleapis.com/youtube/v3/search?part=snippet&type=video&eventType=live&q=${query}&key=${apiKey}&maxResults=50&order=viewCount`
     - Paginate via `nextPageToken` until no more pages or 500 cap
     - Map `items[].id.videoId` to URL, `items[].snippet.title/channelTitle/thumbnails.medium.url` to fields
     - On 403 (quota exceeded), return `ProviderResult` with error `{ type: 'rate_limited', message: '[YouTube] Data API quota exceeded' }`
     - Console log: `[YouTube] Data API search for "${query}": ${count} results`
   - `onDestroy()`: set innertube to null

3. Update `packages/streamwall/src/main/discovery/providers/index.ts`:
   - Import `YouTubeProvider` and instantiate it
   - Export `const providers: BaseProvider[] = [new YouTubeProvider()]`

4. Add search query persistence to `packages/streamwall/src/main/discovery/settings.ts`:
   - Add `searchQuery: string` to `DiscoveryStoreSchema` with default `''`
   - Export `getSearchQuery(): string` and `setSearchQuery(query: string): void`

5. Add CLI flags to `packages/streamwall/src/main/index.ts` in `parseArgs()`:
   - Add a `'Discovery'` option group with `--discovery-query` (string, default '') and `--youtube-api-key` (string, default '')
   - In `main()`, after parsing args:
     - If `argv['discovery-query']` is non-empty, call `setSearchQuery(argv['discovery-query'])`
     - If `argv['youtube-api-key']` is non-empty, call `setApiKey('youtube', argv['youtube-api-key'])`
   - After the discovery process is configured, if there's a persisted search query, send a `search` message:
     ```typescript
     const searchQuery = getSearchQuery()
     if (searchQuery) {
       port2.postMessage({ type: 'search', query: searchQuery })
     }
     ```

Use `node-fetch` (already installed) for the Data API HTTP calls. Follow Prettier conventions (no semicolons, single quotes, trailing commas).
