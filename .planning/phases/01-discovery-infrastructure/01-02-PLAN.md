    - getApiKey decrypts via safeStorage.decryptString from base64
    - Falls back to plain text storage when safeStorage.isEncryptionAvailable() returns false
    - resetDefaults() restores all settings to defaults

    1. Create `packages/streamwall/src/main/discovery/bridge.ts`:
       - Function `createDiscoveryBridge(port: Electron.MessagePortMain): AsyncGenerator<StreamData[]>`
       - Implementation follows RESEARCH.md Pattern 2 (Repeater bridge matching LocalStreamData.gen()):
         ```
         new Repeater(async (push, stop) => {
           await push([])  // initial empty state
           const handler = (event) => {
             if (event.data.type === 'streams') {
               const streams = event.data.payload.map(toStreamData)
               push(streams)
             }
           }
           port.on('message', handler)
           await stop
           port.off('message', handler)
         })
         ```
       - Import `toStreamData` from mapper and `Repeater` from @repeaterjs/repeater
       - Import types: `WorkerOutMessage`, `DiscoveredStream`

    2. Create `packages/streamwall/src/main/discovery-worker.ts` (utility process entry):
       - Entry point for `utilityProcess.fork()`
       - Listens on `process.parentPort` for initial message containing MessagePort
       - On receiving port, listens for `WorkerInMessage` messages:
         - `configure`: stores settings, initializes/updates providers
         - `search`: calls DiscoveryManager.searchAll(query), posts results back as WorkerOutMessage
         - `pause`: sets paused flag, stops polling loop
         - `resume`: clears paused flag, restarts polling loop
         - `destroy`: calls destroyAll, exits cleanly
       - Polling loop: when not paused, calls searchAll at discoveryIntervalMs, sends results per-platform as WorkerOutMessage
       - On unhandled error: logs to stderr, sends error status message
       - Import providers from providers/index.ts (empty array in Phase 1)

    3. Update `packages/streamwall/forge.config.ts`:
       - Add utility process build entry to the VitePlugin `build` array:
         ```typescript
         {
           entry: 'src/main/discovery-worker.ts',
           config: 'vite.main.config.ts',
           target: 'main',
         }
         ```
       - Position it after the main entry but before preload entries

    4. Update `packages/streamwall/src/main/index.ts` to wire discovery into the pipeline:
       - Import: `utilityProcess`, `MessageChannelMain` from 'electron'
       - Import: `createDiscoveryBridge` from './discovery/bridge'
       - Import: `discoveryStore`, `getDiscoverySettings` from './discovery/settings'
       - Import: `markDataSource` (already imported)
       - Inside `main()` function, after existing initialization but before `dataSources` array:
         a. Create MessageChannelMain: `const { port1, port2 } = new MessageChannelMain()`
         b. Fork utility process: `const discoveryProcess = utilityProcess.fork(path.join(__dirname, 'discovery-worker.js'))`
         c. Transfer port1 to utility process: `discoveryProcess.postMessage({ type: 'init' }, [port1])`
         d. Start port2: `port2.start()`
         e. Send initial settings: `port2.postMessage({ type: 'configure', settings: getDiscoverySettings() })`
         f. Subscribe to settings changes: `discoveryStore.onDidAnyChange(() => { port2.postMessage({ type: 'configure', settings: getDiscoverySettings() }) })`
         g. Handle utility process crash with exponential backoff (1s, 2s, 4s, max 5 retries):
            ```
            let restartCount = 0
            discoveryProcess.on('exit', (code) => {
              if (code !== 0 && restartCount < 5) {
                const delay = Math.min(1000 * Math.pow(2, restartCount), 4000)
                restartCount++
                setTimeout(() => { /* re-fork and re-wire */ }, delay)
              }
            })
            ```
         h. Create bridge: `const discoveryBridge = createDiscoveryBridge(port2)`
       - Add to dataSources array: `markDataSource(discoveryBridge, 'discovery')` (note: per-stream _dataSource is overridden by mapper to 'discovery:{platform}')
       - The utility process starts immediately on app launch per user decision

    Note on path resolution: Use `path.join(__dirname, 'discovery-worker.js')` -- Vite resolves __dirname correctly for both dev and packaged builds when the entry has `target: 'main'` in forge config.
