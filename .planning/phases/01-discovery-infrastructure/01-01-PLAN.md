<task type="auto" tdd="true">
  <name>Task 2: Vitest setup, rate limiter, LRU cache, and mapper with tests</name>
  <files>packages/streamwall/package.json, packages/streamwall/vitest.config.ts, packages/streamwall/src/main/discovery/rate-limiter.ts, packages/streamwall/src/main/discovery/lru-cache.ts, packages/streamwall/src/main/discovery/mapper.ts, packages/streamwall/src/main/discovery/__tests__/rate-limiter.test.ts, packages/streamwall/src/main/discovery/__tests__/lru-cache.test.ts, packages/streamwall/src/main/discovery/__tests__/mapper.test.ts</files>
  <behavior>
    Rate limiter (TokenBucket):
    - tryConsume() returns true when tokens available, false when exhausted
    - Bucket starts full (maxTokens available)
    - After maxTokens calls to tryConsume(), next call returns false
    - After refill interval passes, tokens are replenished
    - msUntilRefill() returns positive value when tokens exhausted, 0 when tokens available

    LRU cache:
    - get() returns undefined for missing keys
    - get() returns stored value for existing keys
    - set() stores key-value pairs retrievable by get()
    - When cache exceeds maxSize, oldest (least recently used) entry is evicted
    - get() promotes entry to most-recently-used (prevents eviction)
    - has() returns true for existing keys, false for missing
    - size property reflects current number of entries
    - clear() removes all entries

    Mapper (toStreamData):
    - Converts DiscoveredStream to StreamData
    - Sets _dataSource to 'discovery:{platform}'
    - Maps url -> link, title -> label, channelName -> source
    - Sets kind to 'video'
    - Preserves thumbnailUrl, viewerCount, platform, channelName as optional fields
    - Sets _id to empty string (StreamIDGenerator assigns it later)
  </behavior>
  <action>

    5. Create `packages/streamwall/src/main/discovery/lru-cache.ts` implementing a simple Map-based LRU cache. Constructor takes maxSize. Methods: get(key), set(key, value), has(key), delete(key), clear(). Property: size. Use Map iteration order (insertion order) for eviction -- delete and re-insert on access to move to end.

    6. Create `packages/streamwall/src/main/discovery/mapper.ts` with `toStreamData(d: DiscoveredStream): StreamData` function.
